# Отчёт о найденных багах

В ходе анализа кодовой базы были выявлены следующие проблемы:

1.  **Избыточная точка входа в `userbot.py`**:
    Файл `userbot.py` содержит функцию `main()` и блок `if __name__ == "__main__":`, которые дублируют функционал официальной точки входа `main.py`. Это может привести к путанице и ошибкам при запуске.

2.  **Блокирующие вызовы `input()` в `try_login` (`userbot.py`)**:
    Функция `try_login` использует `input()` для получения номера телефона и SMS-кода. В асинхронном приложении это блокирует весь event loop, что недопустимо, особенно при работе Web UI.

3.  **Отсутствие обработки ошибок при приведении типов в `on_packet` (`userbot.py`)**:
    Вызовы `int(chat_id)` и `int(message_id)` в `on_packet` могут вызвать `ValueError`, если пакет содержит некорректные данные, что приведет к падению обработчика для конкретного сообщения.

4.  **Отсутствие поля `sms_token` в `ActiveAccount` (`core/multiaccount.py`)**:
    В dataclass `ActiveAccount` отсутствует поле для хранения токена SMS-авторизации, хотя методы `send_code` и `login_by_sms` пытаются его использовать через `active.sms_token`.

5.  **Неполный интерфейс Web UI для аутентификации аккаунтов**:
    В текущей реализации Web UI позволяет добавить аккаунт, но не предоставляет возможности отправить SMS-код или ввести полученный код для завершения авторизации.
