# Web UI: Документация модулей и управление конфигами

## 🎯 Новые возможности Web UI

### 1. 📖 Документация модулей в интерфейсе

Теперь каждый модуль в Web UI отображает:
- **Описание модуля** — краткая информация о назначении
- **Список команд** — все доступные команды с алиасами и описанием
- **Конфигурация** — интерактивные поля для настройки

### 2. ⚙️ Управление конфигами через Web UI

#### Как это работает:
1. Откройте Web UI: `http://127.0.0.1:8088`
2. Найдите нужный модуль в сетке
3. В секции "Конфигурация" измените значения полей
4. Нажмите **Save** — конфигурация сохранится автоматически

#### Пример конфига модуля:
```python
BotModule(
    name="Echo",
    description="Повторяет сообщения",
    default_config={
        "enabled": True,
        "prefix": "!",
        "log_channel": "-1001234567890"
    }
)
```

В Web UI отобразятся поля:
- `enabled` → True/False
- `prefix` → строка
- `log_channel` → ID канала

### 3. 🎨 Улучшенный интерфейс

- **Карточки модулей** с четкой структурой
- **Подсветка кода** для команд (моноширинный шрифт)
- **Разделители** между секциями
- **Hover-эффекты** на кнопках
- **Адаптивная сетка** (auto-fill)

### 4. 🔄 Синхронизация конфигов

- Конфиги из Web UI синхронизируются с `userbot_config.json`
- Команды `.config` и `.fconfig` работают параллельно
- Приоритет: последнее изменение (веб или чат)

## 🔧 Технические детали

### Обработка конфигов в коде:
```python
# userbot.py::_module_panel()
module_conf = self.config_store.data.module_configs.get(module.name.lower(), {})
default_conf = module.default_config or {}
display_conf = {**default_conf, **module_conf}
```

### API эндпоинт:
- **POST `/api/config`**
  - Параметры: `module`, `key`, `value`
  - Сохраняет в `config.module_configs[module][key] = value`
  - Перезагружает страницу с обновленными данными

## 🚀 Использование в своих модулях

Пример создания модуля с конфигом:

```python
from userbot import BotModule, ModuleCommand

def setup(registry):
    registry.register_module(
        BotModule(
            name="MyModule",
            description="Мой крутой модуль",
            commands=[
                ModuleCommand(name="start", description="Запустить"),
                ModuleCommand(name="stop", description="Остановить")
            ],
            default_config={
                "auto_start": True,
                "delay": 5,
                "message_template": "Hello!"
            },
            builtin=False
        )
    )
```

После загрузки модуля в Web UI автоматически появятся поля для настройки!
